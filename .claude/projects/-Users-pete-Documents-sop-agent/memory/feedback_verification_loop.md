Stage 3 SOP 생성 후 검증 루프(실제 대화로 SOP 테스트)가 효과적임을 확인.

**Why:** 초안 SOP는 분석 시 읽은 대화 패턴만 커버하므로, 분석에 안 쓰인 대화에서 20~30% 시나리오가 누락됨. 긴 프롬프트로 품질을 강제하는 것보다, 실제 대화 테스트 → 보강 사이클이 더 효과적.

**How to apply:**
- SOP 생성 후 반드시 검증 루프 실행
- 검증 대화 수 = Case 수 × 2 (Case 4개 → 8건)
- 검증에 사용하는 대화는 분석 시 안 읽은 대화 사용 (일반화 테스트)
- 사용자가 "좋네요"로 승인함 (2026-04-07)
